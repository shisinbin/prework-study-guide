HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • HTML attributes are defined in the beginning element, never the closing element.
  • Section elements are semantic elements that define a section in the document that contains thematic content.
  • The id attribute is used as a way to identify elements and select them.

CSS

  • There are three ways of adding CSS: inline, internal, external. The most common is the latter.
  • CSS rules follow a standard pattern/formula: selector, property, value.
  • Kinda linked with HTML, but important for CSS: every HTML element is essentially a box, with a margin, a border and padding, and these things can be fiddled with via using CSS.
  • Two or more elements (e.g. h1, footer,...) can be shared which is a good way to reduce clutter, encourages reusability.
  • Class attributes allow us to share a CSS rule to any element we choose by assigning the CSS rule to a class attribute with a class selector. Boy, that's kinda confusing, but yeah, it's just a way to get things looking a certain way within a sort of section.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git checkout branch-name: switches to branch
  • git remote prune origin OR git fetch -p: removes 'stale' branches on the remote/github side of things
  • git branch -d branch-name: removes the branch locally
  • git branch -r: shows remote (github) branches
  • git branch -a: shows both remote and local branches

JavaScript

  • When we connect a JavaScript file to an HTML file, we stick it in the footer element in the body with script tags.
  • JavaScript code runs from in a logical, top-down way.
  • Being able to store groups of data in an array is a fundamental feature of JavaScript.
  • If conditional statements, for loops, and enveloping bits of code in functions before calling them are common procedures/methods found in JavaScript files.
  • Using the console.log() method and then using Google Chrome's dev tools and console is a useful way of debugging JavaScript code easily.
