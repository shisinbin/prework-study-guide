CSS

  • there are three ways of adding CSS: inline, internal, external
  • css rules follow a standard pattern/formula: property, value
  • kinda linked with HTML, but important for CSS, that every HTML element is essentially a box, with a margin, a border and padding, and these things can be fiddled with with CSS
  • two or more elements (e.g. h1, footer,...) can be shared which is a good way to reduce clutter
  • class attributes allow us to share a CSS rule to any element we choose by assigning the CSS rule to a class attribute with a class selector. boy, that's kinda confusing, but yeah, it's just a way to get things looking a certain way within a sort of section.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git checkout branch-name: switches to branch
  • git remote prune origin OR git fetch -p: removes 'stale' branches on the remote/github side of things
  • git branch -d branch-name: removes the branch locally
  • git branch -r: shows remote (github) branches
  • git branch -a: shows both remote and local branches
